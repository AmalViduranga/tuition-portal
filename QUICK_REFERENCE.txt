🔐 FIRST LOGIN FLOW
───────────────────────────────────────────────────────────────────
1. Student goes to /login
2. Enters email + temporary password
3. System checks: must_change_password = true?
   YES → Redirect to /change-password?required=true
   NO  → Proceed to dashboard
4. Student sets new password (min 8 chars)
5. must_change_password set to false
6. Redirect to dashboard

  3. Reset email sent
  4. Click link in email → /reset-password?token=...
  5. Set new password (min 8 chars)
  6. Redirect to login

Option B - After First Login:
  - Use /change-password from dashboard (if implemented)
